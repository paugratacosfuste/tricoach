## рџљЂ Getting Started

### Prerequisites

- [Node.js](https://nodejs.org/) v18+ (recommended: install via [nvm](https://github.com/nvm-sh/nvm))
- A [Supabase](https://supabase.com) project
- A [Vercel](https://vercel.com) account (for deployment)
- An [Anthropic](https://console.anthropic.com) API key

### Local Development

```bash
# 1. Clone the repository
git clone <YOUR_GIT_URL>
cd tricoach-ai

# 2. Install dependencies
npm install

# 3. Set up environment variables
cp .env.example .env.local
# Edit .env.local with your keys (see Environment Variables below)

# 4. Run the Supabase SQL migration
# Go to your Supabase Dashboard в†’ SQL Editor в†’ paste supabase-schema.sql в†’ Run

# 5. Start the dev server
npm run dev
```

The app will be available at `http://localhost:5173`.

### Environment Variables

Create a `.env.local` file in the project root:

```env
# Supabase (client-side)
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key

# Anthropic (server-side only вЂ” set in Vercel, not in .env.local for production)
ANTHROPIC_API_KEY=your-anthropic-api-key
```

> **Note:** For local development with `vercel dev`, the `ANTHROPIC_API_KEY` should be in `.env.local`. For production, set it in Vercel's environment variables dashboard.

### Database Setup

Run the SQL schema in your Supabase SQL Editor:

1. Open your [Supabase Dashboard](https://supabase.com/dashboard)
2. Navigate to **SQL Editor**
3. Paste the contents of `supabase-schema.sql`
4. Click **Run**

This creates the following tables with RLS policies:
- `profiles` вЂ” user profile and fitness data
- `training_plans` вЂ” race goals and plan metadata
- `weeks` вЂ” individual training weeks
- `workouts` вЂ” daily workout details and actual data
- `week_feedback` вЂ” end-of-week athlete feedback
